casbolwerk commented Mar 5, 2022

What do you want to do with Hls.js?

I want to stream to a webpage and have the functionality to save part of that stream between the time a recording button has been activated and deactivated.

What have you tried so far?

The demo has recording functionality through dumpfMP4 and createfMP4.

The problem I see is that the recording 'works' when activated from the start of the video, where the MP4 file is the total length of the video but the only available content are the buffers that have actually been loaded. So the saved video might be recognized as a 10 min video, but if you only gave it time to add buffers for a short time, the actual video footage might only be a minute or so.

Besides that, the recording doesn't work at all if the dumpfMP4 is only activated after the video has already started. I assume there is some significance to the first buffer or something, where the data added there is essential for the video to work at all. This is not enough if one would like to download parts of a stream.

There is also the issue #1322, where code is shared that is a lot like the code in the demo but supposedly works for some. One comment #1322 (comment) claims to have solved the issue, but is dependent on the now deprecated FRAG_PARSING_DATA event. What has this event been replaced by? Does this fix also fix the issues in the demo?
